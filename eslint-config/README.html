  • Crie/Adicione no .vscode/settings.json (no na suas configuraes, pois assim ficar no projeto e o time j ter acesso):

React Hooks#

Para que o eslint verifique e aplica a regra correta do react-hooks/exhaustive-deps preciso adicionar seus custom hooks nas regras do .eslintrc, para facilitar esse processo criamos um generator. J adicionados todos os hooks do Houston, mas caso queria adicionar os do seu proejeto:

js
1
// .eslintrc.js <~precisa ser .js
2
const generator = require('@eduzz/eslint-config-houston/generator');
3


4
module.exports = {
5
  extends: ['@eduzz/eslint-config-houston'],
6
  rules: {
7
    'react-hooks/exhaustive-deps': generator.exhaustiveDeps('warn', ['useMyCustomHook'], true)
8
  }
9
};
MtodoParams
exhaustiveDeps1. 'warn' | 'error'
2. Array com seus hooks
3. boolean se deve ou no adicionar os hooks do @eduzz/houston-hooks

Vale ressaltar que os custom hooks devem seguir o padro de que o segundo parametro seja a deps

React 17 e JSX#

Para utilizar a nova verso do React com jsx-runtime basta seguir o tutorial do blog, mas resumidamente :

bash
1
# Removendo Imports React no Utilizadas
2
npx react-codemod update-react-imports

tsconfig.json

json
1
{
2
  //... suas configuraes
3
  "compilerOptions": {
4
    "jsx": "react-jsx" //Troque esse configurao
5
  }
6
}

.eslintrc

json
1
{
2
  "extends": ["@eduzz/eslint-config-houston"],
3
  "rules": {
4
    //Adicione essas rules
5
    "react/jsx-uses-react": "off",
6
    "react/react-in-jsx-scope": "off"
7
  }
8
}
